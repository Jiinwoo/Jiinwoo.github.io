## 비교를 통한 배움

여러가지 언어를 배울 때 어떤점이 공통적으로 사용되고 어느 부분이 어떻게 다른지를 알게 된다면 다른 언어를 배울 때도 쉽게 익힐 수 있다고 한다.
책의 예제에서는 참 거짓 값을 C, Ruby, Java 예시를 들었다.

나는 현재 타입스크립트, 코틀린을 배우고 있으니 이 언어들의 참 거짓 값을 알아보자. 
###typescript (javascript)
```typescript
if(undefined || null || 0 || -0 || NaN || "") {

}else {
  
}
```

javascript는 위의 7개 값이 if문 안으로 들어 갔을 때 false가 된다. 

### kotlin(java)
```kotlin
fun main() {
    println("Hello, world!!!")
    var boo: Boolean = true
    if(boo) {
        println("asdasd")
    }
}
```
사실 코틀린은 자바 상위 호환이기 때문에 boolean 타입 외에는 컴파일이 안된다.


## 역사를 통한 배움

언어가 가진 각 기능들은 어떠한 문제들을 해결하려고 나왔다.
역으로 생각해 보면 어떠한 흐름을 따라 만들어졌는지 알게 된다면 해당 언어의 기능을 보다 자세히 알 수 있을것이다.


## 프로그래밍 언어 탄생의 역사

1946년 에니악, 에드삭이 나왔을 때 진공관을 일일히 연결하거나 종이에 구멍을 뚫는 등의 '기계어'를 사용했다. 하지만 이 기계어는 사람이 읽기에
불편했으며 처리할 수 있는 데이터의 양도 적었다.

### FORTRAN

요즘 우리가 일반적으로 쓰는 x + y 같은 표현이 처음 사용되었던 언어가 포트란이다. 비록 초기에는 능숙한 프로그래머가 짠 기계어에 비해 효율이 
떨어졌지만 코드량이 줄어듬과 동시에 읽기 쉬워짐에 따라 많이 사용되었다. 
 
 ### 프로그램의 삼대미덕 중 하나인 게으름
 앞날 까지 생각해서 가장 편한 방법을 선택하는 것, 즉 같은 성과를 달성하는 다수의 방법 중 가장 생산성이 높은 것을 선택함.
 
 ### 무엇을 편하게 하고 싶은 것인가?, 어떤 프로그램을 편하게 만들고 싶은가?
 모든 언어의 존재 목적은 '편리함' 이라고 했지만 각 언어가 추구하는 '편리함'은 서로 다르다. 
 
 * 빠른 실행속도 : c++
 * 언어의 사양을 쉽게 파악 : Scheme
 * 다른사람이 쓴 코드를 쉽게 해석할 수 있도록 : Python
 * 웹서비스 : PHP
 * 문장 처리 : Haskell, OCaml
